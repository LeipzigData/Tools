Leipzig Data Event Widgets

Mit den Leipzig Data Event Widgets werden die prinzipiellen Möglichkeiten des Leipzig Data Events Frameworks demonstriert.

Unterschiede der Widget Versionen

Es gibt 2 verschiedene Widget Versionen. Sie benutzen momentan die umfangreichere Full Widget Version.
Die Versionen unterscheiden sich in der Art und Weise, wie Daten zur Präsentation eingespielt werden.
  • Simple Widget
    Das Simple Widget baut eine direkte Verbindung zu einem SPARQL-Endpunkt auf (z.B. dem von leipzig-data.de) und bezieht anhand einer nutzerdefinierten Anfrage Daten aus diesem. Es ist sehr leichtgewichtig und verwendet keinen eigenen Triplestore. Es benötigt somit auch keine eigene MySQL-Datenbank und verzichtet auf weitere Frameworks.
  • Full Widget
    Das komplexere Widget verfügt über einen eigenen Triplestore. In diesen können RDF-Daten wie ein von http://leipzig-data.de/Data heruntergeladener Dump oder eigene Daten eingespielt werden. Zusätzlich können weitere Daten aus anderen Quellen eingespielt und mit den vorhandenen aggregiert werden. Ein Filtermechanismus erlaubt es anschließend, nur bestimmte Daten für eine Präsentation auszuwählen. Als besonderes Feature stellt dieses Widget noch einen öffentlich zugänglichen und konfigurierbaren SPARQL-Endpunkt für die aggregierten Daten bereit. Das Widget benötigt im Gegensatz zur Simple Version eine MySQL Datenbank und verwendet das ARC2-Framework.

Einspielen der Daten

  1. Herunterladen eines aktuellen Dumps mit Event-Daten im Turtle-Format als EventsDump.ttl in dieses Verzeichnis. Bei Bedarf können weitere Dateien eingespielt werden, indem die Ladeanweisungen der Form
    $store->query("LOAD <file:Datei.ttl>");
    in der Datei Store.php entsprechend ergänzt werden.
  2. Laden aller Daten in den Store mittels Aufruf von Store.php.
  3. Erzeugen der Präsentationdsatei und ggf. Filtern der Daten durch angepasste SPARQL-Anfrage mittels getdata.php.

Präsentation der Daten

Auf den folgenden drei Webseiten werden die eingespielten Daten als
  • eine einfache Präsentation mit Karte, Eventliste, Kalender und Tagwolke,
  • eine komplexe Präsentation mit verschiedenen stabilen Facetten, Filter- und Suchmöglichkeiten sowie
  • eine experimentelle Präsentation mit noch genauer zu fixierenden Facetten
dargestellt.
  • Technische Dokumentation
Die Präsentation ist Teil des Leipzig Open Data Projekts.
